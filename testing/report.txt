1. ΡΥΘΜΙΣΗ ΒΑΣΗΣ ΔΕΔΟΜΕΝΩΝ
----------------------------------------------------------------
Βήμα 1: Εκκίνηση XAMPP
  Ανοίξτε τον XAMPP Control Panel και εκκινήστε τις υπηρεσίες
  Apache και MySQL.

Βήμα 2: Δημιουργία Βάσης Δεδομένων
  Ανοίξτε τον browser και μεταβείτε στη διεύθυνση:
    http://localhost/phpmyadmin

  Στο phpMyAdmin:
    - Επιλέξτε "Import" από το επάνω μενού
    - Κάντε κλικ στο "Choose File"
    - Επιλέξτε το αρχείο setup.sql από τον φάκελο backend
    - Κάντε κλικ στο "Go"

  Αυτό δημιουργεί αυτόματα τη βάση δεδομένων
  "city_problem_reporting" με όλους τους απαραίτητους πίνακες
  (users, categories, reports).


2. ΕΓΚΑΤΑΣΤΑΣΗ & ΕΚΚΙΝΗΣΗ
----------------------------------------------------------------
Βήμα 1: Εγκατάσταση εξαρτήσεων Backend
  Ανοίξτε τερματικό (Command Prompt ή PowerShell), μεταβείτε
  στον φάκελο ERGASIA και εκτελέστε:

    > npm install

  ΣΗΜΑΝΤΙΚΟ: Ακόμα και αν ο φάκελος node_modules υπάρχει ήδη,
  συνιστάται να τον διαγράψετε πρώτα και να εκτελέσετε εκ νέου
  npm install. Η βιβλιοθήκη bcrypt χρησιμοποιεί μεταγλωττισμένο
  (compiled) κώδικα που είναι δεμένος με τον υπολογιστή στον
  οποίο εγκαταστάθηκε. Αν αντιγραφεί από άλλον υπολογιστή, δεν
  θα λειτουργεί και θα εμφανίζεται "Internal server error" κατά
  τη σύνδεση ή εγγραφή χρηστών.

Βήμα 2: Δημιουργία λογαριασμού διαχειριστή
  Εκτελέστε:

    > node seed.js

  Στοιχεία σύνδεσης διαχειριστή:
    Email   : admin@gmail.com
    Κωδικός : admin123

Βήμα 3: Εκκίνηση Server
  Εκτελέστε:

    > node app.js

  Ο server θα εκκινήσει στη διεύθυνση: http://localhost:3000

Βήμα 4: Πρόσβαση στην Εφαρμογή
  Ανοίξτε τον browser και μεταβείτε στη διεύθυνση:
    http://localhost:3000

  Το frontend φορτώνεται αυτόματα από τον ίδιο server.


3. ΑΛΛΑΓΕΣ ΠΟΥ ΠΡΑΓΜΑΤΟΠΟΙΗΘΗΚΑΝ ΣΤΟ BACKEND
----------------------------------------------------------------
Κατά την ανάλυση του κώδικα εντοπίστηκαν τρεις ελλείψεις που
εμπόδιζαν την πλήρη λειτουργία της εφαρμογής.

-----
ΑΛΛΑΓΗ 1 — controllers/report.js
           Προσθήκη συνάρτησης επεξεργασίας αναφοράς
-----
Το αρχικό αρχείο δεν περιείχε καμία συνάρτηση για την
επεξεργασία υφιστάμενης αναφοράς. Όταν ένας πολίτης
προσπαθούσε να επεξεργαστεί αναφορά με κατάσταση "NEW",
η εφαρμογή επέστρεφε σφάλμα.

Προστέθηκε η συνάρτηση editReport η οποία:
  - Ελέγχει ότι η αναφορά ανήκει στον συνδεδεμένο χρήστη
  - Επαληθεύει ότι η κατάσταση είναι "NEW"
    (μόνο νέες αναφορές επιτρέπεται να επεξεργαστούν)
  - Επικυρώνει τα δεδομένα εισόδου με Zod
  - Ενημερώνει τα πεδία category_id, address και description
    στη βάση δεδομένων

-----
ΑΛΛΑΓΗ 2 — routes/report.js
           Προσθήκη HTTP διαδρομής PATCH
-----
Ακόμα και με την ύπαρξη της συνάρτησης editReport, ο Express
server δεν γνώριζε πότε να την καλέσει, διότι δεν υπήρχε η
αντίστοιχη HTTP διαδρομή (route).

Προστέθηκε η διαδρομή:
  PATCH /api/reports/:id  →  editReport
με τους κατάλληλους ελέγχους αυθεντικοποίησης και ρόλου
(citizen).

-----
ΑΛΛΑΓΗ 3 — app.js
           Σερβίρισμα του Frontend από τον Backend Server
-----
Το αρχικό app.js δεν περιείχε κώδικα για την εξυπηρέτηση των
στατικών αρχείων του frontend. Η εφαρμογή δεν ήταν προσβάσιμη
μέσω browser από τον backend server.

Προστέθηκε κώδικας που:
  - Ελέγχει αν υπάρχει ο φάκελος dist/ δίπλα στο app.js
  - Αν ναι, σερβίρει τα στατικά αρχεία του React frontend
  - Ανακατευθύνει όλες τις μη-API διευθύνσεις στο index.html,
    ώστε το React Router να λειτουργεί σωστά

Έτσι, ο φάκελος dist/ τοποθετείται μέσα στον φάκελο backend
και η εφαρμογή σερβίρεται αυτόματα στο http://localhost:3000.
